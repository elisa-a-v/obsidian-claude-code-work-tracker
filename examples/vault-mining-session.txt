‚óŹ That's a rich haul. Here's what I'd distill as the high-value institutional
  knowledge that lives in your vault but not in the CL repo's CLAUDE.md:

  1. The Two-Stack Trap

  Every template is either legacy (base.html) or new (new_base.html). New-stack 
  tools silently fail in legacy templates ‚ÄĒ Tailwind classes don't render,
  Alpine breaks, Cotton components don't work. A merged PR already broke things
  this way. Always check {% extends %} before touching a template.

  2. CSP-Friendly Alpine (the #1 gotcha)

  CL uses the CSP build of Alpine. Almost all Alpine documentation examples will
   not work ‚ÄĒ no inline expressions, no x-data="{ open: false }", no
  @click="thing = !thing". All logic goes in external JS files loaded via {% 
  require_script %}. Also: x-model and x-modelable are unsupported (require
  unsafe-eval).

  3. Semantic HTML Preferences

  - <dl> for key-value/metadata, not <table> or <ol> ‚ÄĒ this gets flagged in
  every review
  - <ul> can't directly nest inside <ul> ‚ÄĒ must be inside <li> (flagged multiple
   times)
  - Prefer <details>/<summary> for collapsible content (progressive enhancement)

  4. Accessibility as a Core Value

  - Target: WCAG 2.2 AA minimum, AAA where feasible
  - Keyboard nav regressions are taken very seriously ‚ÄĒ when rewriting
  jQuery/React to Alpine, you must preserve arrow key nav, Escape behavior, 
  focus management
  - Dynamic content needs aria-live regions
  - Links must be recognizable without color alone (external = underline or
  icon)

  5. Real Bugs from Pattern Violations

  - Cotton :attr="{{ var }}" is wrong ‚ÄĒ correct is :attr="var" (no DTL braces)
  - {{ disclosure.filepath }} gives storage-relative paths, not URLs ‚ÄĒ must use
  .url
  - data-first-active hardcoded to an ID that didn't exist in nav_items
  - CounterMetricFamily strips _total suffix ‚ÄĒ test assertions need the base
  name
  - Prometheus metrics in Celery tasks are invisible (process-local memory,
  never scraped) ‚ÄĒ use tally_stat() or fetch-on-scrape pattern

  6. PR Review Culture

  - Screenshots must be taken after code is finalized, not before
  - Don't duplicate existing components ‚ÄĒ extend them
  - Don't add {% include %} in v2 templates ‚ÄĒ use Cotton
  - Don't add nofollow to editorial links (that's for user-generated content)
  - Don't create single-use utility classes when inline Tailwind works
  - Legacy templates need sync notices when a v2 counterpart exists

  7. Redesign Strategic Context

  - Help pages are paused ‚ÄĒ may be replaced by a vibe-coded wiki
  - Profile pages are next priority
  - Docket pages have 10 implementation issues filed
  - Prioritization shifted from "simplicity-first" to "impact over simplicity"
  - Tailwind v3 (not v4) ‚ÄĒ legal users keep devices outdated

  section of the CL repo's CLAUDE.md or a linked reference doc. That way any
  agent working in the repo gets it automatically. Want me to draft that?
